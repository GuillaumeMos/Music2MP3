# Audit 2026-05-26 - SoundCloud et scope produit

## Constat

- Les tests unitaires passaient, mais ils ne couvraient pas le lancement reel de `yt-dlp` depuis le venv.
- `SoundCloudClient` resolvait `yt-dlp` en renvoyant seulement `yt-dlp` ou `yt-dlp.exe`.
- Dans l'environnement courant, l'executable existe dans `.venv/bin/yt-dlp`, mais ce dossier n'est pas forcement dans le `PATH` quand l'app est lancee via `.venv/bin/python qt_app.py`.
- Resultat: l'import SoundCloud echouait avant l'appel reseau avec `FileNotFoundError: [Errno 2] No such file or directory: 'yt-dlp'`.
- Le test live direct via `python -m yt_dlp` montre aussi des erreurs SoundCloud 403 pour certains items, mais `yt-dlp` peut quand meme retourner le JSON de playlist. Le bon comportement applicatif est donc de lancer `yt-dlp` correctement, puis de remonter les 403 reels avec le message Browser auth/cookies deja prevu.

## Correctifs appliques

- Ajout d'un helper unique `find_ytdlp_cmd()` dans `utils.py`.
- Resolution `yt-dlp` par priorite:
  1. binaire bundle avec l'app,
  2. executable voisin de l'interpreteur courant, typiquement `.venv/bin/yt-dlp`,
  3. executable disponible dans le `PATH`,
  4. fallback module Python courant: `sys.executable -m yt_dlp`.
- `soundcloud_api.py` utilise maintenant ce prefixe de commande, ce qui repare le lancement depuis le venv.
- Pour les playlists SoundCloud (`/sets/`), `soundcloud_api.py` lit maintenant d'abord la page publique via `window.__sc_hydration`; ca evite d'attendre `yt-dlp` pendant plus d'une minute avant un 403.
- Si ce fallback page publique ne donne rien, le client retente ensuite `yt-dlp` full puis flat.
- `bandcamp_api.py` utilise le meme helper pour garder un comportement coherent, meme si Bandcamp est retire du chemin actif.
- Ajout d'un test de fallback module Python.
- Bandcamp et Soulseek/slskd sont parques en backlog:
  - boutons caches/desactives dans la sidebar,
  - settings slskd non exposes,
  - bouton "Search Soulseek" retire du detail d'erreur,
  - manifests Bandcamp non syncables,
  - `slskd-api` retire des requirements actifs.
